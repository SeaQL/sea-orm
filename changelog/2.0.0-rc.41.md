## Release Notes: SeaORM 2.0.0-rc.41

*(pending)*

### Enhancements

**Stable serde for first-party value wrappers** (#3023)

SeaORM now provides explicit serde implementations for first-party value
wrappers instead of relying on tuple/newtype derive shape.

`TextUuid` serializes as a UUID string and deserializes by parsing that string.
Unix timestamp wrappers serialize as the same `i64` value used for database
storage:

- `ChronoUnixTimestamp`
- `ChronoUnixTimestampMillis`
- `TimeUnixTimestamp`
- `TimeUnixTimestampMillis`

**Migration CLI custom connection entrypoints** (#3035)

`sea-orm-migration` now exposes custom connection entrypoints for the migration
CLI. This allows applications to customize `ConnectOptions`, load SQLite
extensions, or construct a connection from application-specific configuration
before running migration commands.

**Generated migration names** (#2580)

Generated migration files now implement `MigrationName` explicitly, so migration
names are stable even when file/module layout is customized.

**Entity loader diamond relations** (#3030)

Entity loader now distinguishes multiple relations to the same target entity by
their explicit relation definition instead of matching only by table reference.
This fixes diamond-shaped relation graphs where two fields point to the same
entity through different foreign keys.

**Explicit prefixes for nested result models** (#2989)

`FromQueryResult` and `DerivePartialModel` now support
`#[sea_orm(nested(prefix = "..."))]`, allowing applications to explicitly choose
the column prefix used when decoding nested result models. This is useful when a
result model contains multiple nested fields of the same Rust type.

**Streaming behind `stream` cargo feature** (#3075)

Streaming APIs (`QueryStream`, `StreamTrait`, `Select::stream*`) are now gated
behind the `stream` cargo feature. `stream` is included in the default features
of both `sea-orm` and `sea-orm-sync`, so default builds are unaffected.
`default-features = false` builds that do not enable `stream` drop the
`ouroboros` dependency and hide the streaming APIs.

### Bug Fixes

**Schema-sync indexes for non-default PostgreSQL schemas** (#3085, #3084)

Indexes generated from `#[sea_orm(indexed)]` and `#[sea_orm(unique_key = "...")]`
on an entity with a `schema_name` now target the schema-qualified table on
PostgreSQL (e.g. `ON "sys"."app_user"` instead of `ON "app_user"`), so
schema-sync no longer fails with `relation "..." does not exist` or targets a
same-named table in the wrong schema. MySQL and SQLite keep using unqualified
index targets (SeaQuery's index builders for those backends do not accept a
schema-qualified target), which also fixes a latent panic when syncing a
`#[sea_orm(unique)]` column on a `schema_name` entity against MySQL/SQLite.

**SQLite file URI parameters** (#3072)

Rusqlite connections now allow SQLite file URI query parameters and leave URI
parameter validation to SQLite. This fixes valid file URI forms such as shared
in-memory databases with query parameters.

**SQLite timestamp with time zone reads** (#2878)

SQLite timestamp-with-time-zone values now preserve the stored offset instead of
normalizing through UTC when read back.

**CLI extra attribute parsing** (#3032)

SeaORM CLI entity generation now handles comma-separated extra attributes and
derives that contain nested commas, such as:

```shell
--model-extra-attributes 'serde(rename_all = "camelCase"),ts(export)'
```

**PostgreSQL drop everything with dependent custom types** (#3014)

`sea-orm-migration` now drops PostgreSQL custom types with `CASCADE` during
`drop_everything`, allowing dependent domains and functions to be cleaned up.
Extension-owned types are still preserved.

**Generated `delete_by_*` for single-column unique keys** (#3059)

`model_ex` now generates `delete_by_*` helpers for one-column unique keys,
matching the existing generated `find_by_*` behavior for `#[sea_orm(unique)]`
and single-column `unique_key` fields.

**Tracing instrumentation no longer records SQL arguments by default** (#3047)

Automatic `tracing::instrument` fields no longer record SQL arguments. SQL
statement recording remains controlled by the explicit tracing span setting.

- Added regression coverage for generated `delete_by_*` helpers on
  single-column unique keys.

### Compatibility Notes

The serde support changes JSON shape for first-party wrappers to explicit,
stable representations:

- `TextUuid` is a JSON string.
- Unix timestamp wrappers are JSON integers.

SQLite timestamp-with-time-zone reads now preserve offsets. PostgreSQL
`drop_everything` is more complete and intentionally more destructive for
objects that depend on custom types.

The entity loader change adds relation-specific loader methods to the public
loader traits. Existing generated code is updated automatically, but manual
implementations of those traits need to add the new methods.

Applications building with `default-features = false` that use streaming APIs
now need to enable the `stream` feature explicitly.

Nested result model prefixes are opt-in. Existing unprefixed
`#[sea_orm(nested)]` fields, including multiple fields with the same nested Rust
type, continue to compile. If multiple fields explicitly use the same non-empty
prefix for the same nested type, the derive now emits a compile error because
those fields would decode from the same column aliases.
